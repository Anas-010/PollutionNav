# Mumbai AQI Real-Time Route Planner

Find the fastest and cleanest route through Mumbai using real-time air quality data.

## Project structure

```
mumbai_aqi_realtime/
├── app.py                   ← Streamlit UI entry point
├── config.py                ← All constants, palettes, API settings
├── requirements.txt
├── data/
│   ├── road_aqi_output.csv          ← From Interpolation_Road_AQI.ipynb (fallback)
│   └── cleaned_aqi_mumbai_imputed.csv ← Station coordinates for map markers
└── modules/
    ├── __init__.py
    ├── openaq_client.py     ← OpenAQ v3 fetch + CPCB AQI computation
    ├── interpolation.py     ← Spatial interpolation surface builder
    ├── routing.py           ← ORS route fetch + pollution exposure calculation
    ├── geocoding.py         ← Nominatim geocoding with Mumbai bias
    └── map_builder.py       ← Folium map construction
```

## Setup

### 1. Install dependencies
```bash
pip install -r requirements.txt
```

### 2. Get API keys (both free)
- **OpenAQ:** https://explore.openaq.org/register
- **OpenRouteService:** https://openrouteservice.org/dev/#/signup

### 3. Place data files
Copy these files (generated by the preprocessing and interpolation notebooks) into the `data/` folder:
- `road_aqi_output.csv` — from `Interpolation_Road_AQI.ipynb`
- `cleaned_aqi_mumbai_imputed.csv` — from preprocessing stage

### 4. Run the app
```bash
streamlit run app.py
```

## How it works

### Live data flow
```
OpenAQ v3 API
  → fetch_live_aqi()        — raw pollutant concentrations for 27 Mumbai stations
  → compute_station_aqi()   — CPCB sub-index formula → AQI per station
  → sample_surface_at_roads() — interpolate AQI at every road segment midpoint
  → [cached 30 minutes]
```

### Route planning flow
```
User types origin + destination
  → geocode()              — Nominatim → (lat, lon)
  → fetch_routes()         — OpenRouteService → up to 3 route alternatives
  → compute_exposure()     — for each route: distance-weighted mean AQI
  → tag_fastest_and_cleanest()
  → display comparison, map, AQI profile chart
```

### Fallback
If fewer than 5 stations return live data (maintenance, API outage, no key),
the app automatically falls back to pre-computed historical averages from
`road_aqi_output.csv`, matched to the current time-of-day and season.

## Data freshness
OpenAQ ingests from CPCB which publishes hourly averages. The pipeline
adds ~1–2 hours of delay. The app clearly communicates this to users.
The AQI surface is re-fetched every 30 minutes.

## Month mapping (for historical fallback)
| Current month | Data used | Reason |
|---|---|---|
| Jan–Feb | December | Dry winter, similar conditions |
| Mar–Apr | October | Post-monsoon dry spell |
| May–Jun | July | Pre-monsoon / monsoon onset |
| Jul–Dec | Exact match | Data available |

## AQI standard
CPCB National Air Quality Index (2014).
AQI = max(sub-index of PM2.5, PM10, NO2, CO, SO2).
Requires at least 3 pollutants with valid readings.
